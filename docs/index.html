What general design systems miss

Six gaps in every AI feature UI

General design systems handle typography, spacing, and colour. They were not built for AI. Streaming, states, ARIA live regions, prompt UX, and AI-specific tokens are outside their scope entirely.

States

Only the success state was designed

Streaming looks broken without a cursor. Error is a blank screen. Empty is a void with a text box. Six of the seven states were never designed — they just happened.

→ ai-states-required
Streaming UI

No streaming cursor. No [Stop] button.

A textarea and a div are not AI-native components. Streaming output needs role="log", a blinking cursor, auto-scroll that respects user position, and live markdown rendering.

→ ai-component-patterns
Accessibility

Screen readers hear nothing during streaming

Tokens injected into the DOM without aria-live="polite" are silent to assistive technology. The streaming cursor needs aria-hidden="true" or it announces on every token.

→ accessible-ai-output
Prompt UX

It's a textarea with Enter to send

A prompt input is not a textarea. It needs auto-resize, Cmd+Enter semantics, a suggestion system, a feedback loop (idle → submitting → streaming → complete), and a token counter.

→ prompt-ux-design
Output Design

Unstyled markdown. No confidence signals.

Rendered markdown without typography is a wall of text. Responses with hedging language look identical to confident responses. There is no progressive disclosure for 2000-word outputs.

→ ai-output-design
Design Tokens

#3b82f6 hardcoded in six components

AI components built under deadline hardcode colour, spacing, and type. Each new component copies the pattern. A rebrand becomes a multi-file migration instead of three token values.

→ design-token-audit

The 8 Skills

One skill per design gap

Pre-implementation gates block implementation before design exists. Pre-ship gates block merge until design is verified.

ai-states-required All 7 states (Loading, Streaming, Success, Error, Partial, Uncertain, Empty) documented before any component code Before Code
design-before-code Spec first — layout, states, copy, interactions — at design/specs/<feature>.md before implementation begins Before Code
accessible-ai-output ARIA live regions, DOM reading order, cognitive load review — 20 minutes before ship, not a week of retrofitting after Before Ship
ai-component-patterns 6 AI-native components — Prompt Input, Streaming Output, Citation Display, Agent Log, Uncertainty Indicator, Empty State — with required sub-components Before Ship
prompt-ux-design Input pattern, suggestion system, 6-state feedback loop, history, token limits — not "just a text box" Before Code
ai-output-design 11 markdown elements, typographic hierarchy, confidence signals, progressive disclosure — defined rendering rules, not "just render the markdown" Before Code
ai-onboarding-design First win, capability communication, trust signals, showcase first response, "model doesn't know" state — not a blank input Before Code
design-token-audit 5 token categories, 5 AI-specific tokens, dark mode verification — zero hardcoded values before merge Before Ship
How enforcement works

A format that cannot be faked

Each skill has a Completion Statement Format that must be filled with real values. An agent cannot claim all 7 states are designed without showing a state map at a real file path.

ai-states-required — completion statement
AI states designed.
Feature: document-assistant
Touchpoints covered: 2 (main response area, inline citation panel)
State map: design/ai-states/document-assistant.md ✓

All 7 states documented:
Loading: skeleton (3 lines), "Searching your documents…", [Cancel] ✓
Streaming: cursor (▋), live text, [Stop], auto-scroll stops on user scroll ✓
Success: full response, [Copy] [👍 👎] [Retry], citation list ✓
Error: red border, "Something went wrong. Try again.", [Try again] ✓
Partial: dashed border, "Response cut off — Continue?", [Continue] ✓
Uncertain: amber left bar, ⚠ "Low confidence — verify this", [Show sources] ✓
Empty: 4 example prompts, input focused, capability boundary stated ✓

Edge cases:
Stream interrupted mid-word: partial token cleared, Error state shown ✓
Empty string response: treated as Partial, not Success ✓
Submit during generation: queued, shown after current completes ✓

An agent that skipped the state design cannot produce design/ai-states/document-assistant.md ✓
with real copy for all 7 states. That is the enforcement.

The 5 Agents

Specialists in AI UI design

Each agent covers one domain. Reviewers produce BLOCK / WARN / NOTE verdicts and write reports to files — not summaries in chat.

Opus

ai-ui-designer

Full AI component design from spec to token-compliant implementation. Writes design/specs/ and design/ai-states/ before any code. Refuses to hardcode any value.

Sonnet

ux-critic

Audits against all 5 skill areas. Writes the critique to design/reviews/<feature>/. Overall verdict: PASS / CONDITIONAL / BLOCK — not "looks good."

Sonnet

state-designer

Maps all 7 states with real copy, real transitions, and 3 edge cases before a single component is designed. Writes to design/ai-states/<feature>.md.

Sonnet

prompt-ux-designer

Chooses the input pattern, designs the suggestion system, maps the 6-state feedback loop, and defines constraints. Never uses Enter-to-send for multi-line input.

Sonnet

accessibility-reviewer

Reviews ARIA, keyboard, contrast, and cognitive load. Writes the report to design/reviews/<feature>/. Will not approve streaming output without role="log" aria-live="polite".

Before and after

What changes

The difference is not catching issues in review. It is the work existing before implementation starts.

✗ Without builder-design

✗ Streaming output is a div. No cursor. Auto-scroll fights the user when they try to read.
✗ Error state is an uncaught exception. Users see a blank screen or a browser error.
✗ Prompt input is a fixed-height textarea. Enter sends. Long messages overflow silently.
✗ Streaming content is invisible to screen readers. No ARIA live region.
✗ #3b82f6 hardcoded in 6 components. Rebrand = migration sprint.
✗ Empty state is a blank text input. First-time users have no idea what to ask.

✓ With builder-design

✓ role="log" aria-live="polite", streaming cursor aria-hidden, scroll follows user position.
✓ Error state: red border, "Something went wrong. Try again.", [Try again] re-focuses input.
✓ Auto-resize textarea, Cmd+Enter sends, [Stop] during generation, token counter at 80%.
✓ Streaming announced as it arrives. Cursor silent. Errors interrupt. Actions reachable by keyboard.
✓ All colours, spacing, shadows reference tokens. Theme change = three token values.
✓ 4 clickable domain-specific prompts. Capability boundary stated. Input focused on load.
